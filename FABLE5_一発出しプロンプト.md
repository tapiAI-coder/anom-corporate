あなたは、世界トップクラスのクリエイティブ・フロントエンドエンジニア兼アートディレクターです。

## 最重要の制約（必ず守る）
- 出力は **単一のHTMLファイル**（`index.html`）。これ単体をブラウザで開けば完全に動くこと。
- ライブラリは **CDN（unpkg / cdnjs / jsdelivr）** から読み込む。ビルド工程・npm・ローカルサーバー不要。
- 推奨ライブラリ: **Three.js**（WebGL演出）、**GSAP + ScrollTrigger**（アニメーション）、**Lenis**（慣性スクロール）。必要に応じて追加可。
- コード内コメントは**日本語**で要点を記す。
- 省略・プレースホルダ（"ここに実装" 等）を残さず、**完成品**を出力する。

## ブランド
- 社名: **ANOM（アノム）** ／ 由来: Autonomous + Minimalism（自律＋簡潔）
- 事業: 秋田県の中小企業向けの DX・AI 導入コンサルティング（戦略から定着まで伴走）
- ミッション: 無駄を削ぎ落とし、AIで自走する組織へ。秋田と関東の差をなくす。
- ターゲット: 秋田の中小企業の経営者（30〜50代）。重視するのは「信頼感・本物感・知性」。

## デザイン方針
- ベースは **白基調のクリーンでミニマルなデザイン**（広い余白・細いライン・整理された情報）。
- そこに**先進的なWebGL/モーション演出**を効かせ、明暗を交互に構成する:
  - **ヒーロー = ダークな面**（ディープネイビー〜ほぼ黒）で発光演出を映えさせる
  - **本文セクション = 明るいクリーン**（白）で可読性と信頼感を担保
  - **最終CTA = 再びダークな面**で締める
- カラー:
  - インク（濃紺）: `#16243E`
  - 背景（白）: `#FFFFFF` ／ 温白（セクション）: `#F7F6F1`
  - 文字（弱）: `#5A6275`
  - **差し色（アクセント）: エレクトリックブルー `#4D5DFF`**（粒子・グロー・モーフィングに使用）
- タイポグラフィ: 英字 **Inter**、日本語 **Noto Sans JP**（Google Fonts）。見出しは大きく、字間を効かせる。
- ロゴは画像がない前提で、`ANOM` を字間を広げたワードマークとして上品に組む。

## 演出の中核コンセプト（重要 — これがこのサイトの背骨）
**「混沌 → 秩序」**。散らばった粒子が集まって `ANOM` の形になる。
= 「無駄（混沌）を削ぎ落とし、自走する組織（秩序）へ」というミッションを視覚で語る。
全演出をこのテーマに沿わせ、**飾りでなく意味のある動き**にすること。

## 採用する技術演出（これらを効果的に組み合わせる。全部を均等に使わず、メリハリをつける）
1. **ヒーロー: WebGL粒子の再構成 + キネティック文字**
   - 数千の粒子が無秩序に漂う → スッと収束して `ANOM` の文字（またはシンボル）を形成する。
   - **カーソル連動**: カーソル付近の粒子が反発/吸着して揺らぐ。
   - 文字組み上げと同期して、キャッチコピーが1文字/1単語ずつ blur→clear で立ち上がる。
2. **慣性スクロール（Lenis）+ スクロール連動の出現**
   - 各セクションのテキストが、スクロールに連動して下から立ち上がる／フォーカスが合う。
3. **見せ場のスクロールズーム/トンネル（サイト内で1回だけ）**
   - ヒーローから本文へ移る瞬間に、「組織の内側へ潜る」ようなズーム/トンネル演出を入れる。多用しない。
4. **セクション間のモーフィングUI（差し色）**
   - アクセントブルーの有機的な塊（ブロブ）が、スクロールに応じて形を変えながらセクションをつなぐ。
   - SVG/Canvas/シェーダーいずれでも可。重すぎない実装にする。
5. **カーソルインタラクション**
   - カスタムカーソル ＋ 主要ボタンの**マグネティック（吸着）効果**（軽い物理感）。
6. **カードの微3Dチルト**
   - About/Services のカードに、ホバーで奥行きのある微小な傾き（parallax tilt）。

不要なもの: 3D商品ビュー（商品がないため）、過剰な物理演算UI（散らかるため避ける）。

## ページ構成と原稿（この文言をそのまま使う）

### 3. なぜANOMか／ラベル `WHY ANOM`

  - 02 一人体制のスピード — 意思決定の階層なし。相談から着手までが短い。

### 4. UseCases／ラベル `02 / USECASES`

  - 建設・土木 — **書類と写真を、AIで軽く**。OCRと業務フロー自動化で事務負担を削減。
  - 小売・飲食 — **売上データを、発注判断に**。売上予測と発注提案で属人化と廃棄ロスを削減。
  - 製造 — **熟練の目を、AIで補完する**。画像AI検査補助とナレッジ検索で若手が自走。
  - サービス・士業 — **問い合わせを、AIと分業する**。一次対応と文書下書きの自動化。

### 5. Services／ラベル `03 / SERVICES`
- 見出し: **ANOMにできること。**

  - **AI・DX顧問（月額）** — 経営・事業判断へのAI/DX相談、ツール選定の中立アドバイス、月次の振り返りと次アクション設計。

## 品質要件
- **レスポンシブ必須**（PC / タブレット / スマホ）。スマホでは重い演出を自動で軽量化（粒子数を減らす等）。
- **パフォーマンス**: 60fps を目標。WebGLは `requestAnimationFrame` ベース、`transform/opacity` 中心。
- **アクセシビリティ**: コントラスト確保、`aria-label`、キーボード操作。`prefers-reduced-motion` 指定時は派手な演出を停止し静的表示にフォールバック。
- 装飾はミッション（無駄を削ぐ）に反しない範囲で。**過剰演出で可読性・信頼感を損なわない**こと。
