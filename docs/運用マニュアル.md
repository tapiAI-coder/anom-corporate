最終更新: 2026-06-11
対象: 菊地さん本人（コード知識がなくても、このマニュアルだけで日常運用が完結することを目指す）

## 1. サイトの全体像（30秒で把握）

- **本番URL**: https://anom-ai.com/
- **仕組み**: GitHubの「anom-corporate」リポジトリの main ブランチにあるファイルが、そのまま公開される（GitHub Pages）。**ファイルを更新→pushすれば数分で本番に反映**
- **サーバー代**: 無料（GitHub Pages）。ドメイン代のみ発生
- **構成**: ビルド不要の素のHTML/CSS/JS。特別なツールなしで編集できる

### どこを直せば何が変わるか

| やりたいこと | 編集するファイル |
|---|---|
| 文言の修正（見出し・説明文・FAQ等） | `index.html` |
| フォームの設定（キー・件名・メール） | `assets/js/config.js` |
| 色・余白・文字サイズ | `assets/css/style.css`（冒頭に目次あり） |
| プライバシーポリシー | `privacy.html` |

## 2. いちばんよく使う作業: 文言修正→公開

### 方法A: Claude Codeに頼む（推奨）
「index.htmlの〇〇という文言を△△に直して公開して」と指示するだけ。
修正→ローカル確認→コミット→pushまで任せられる。

### 方法B: GitHubのWeb画面で直接編集（自分でやる場合）
1. https://github.com/tapiAI-coder/anom-corporate を開く
2. 直したいファイル（例: index.html）をクリック → 鉛筆アイコン（Edit）
3. 文言を修正 → 緑の「Commit changes」ボタン
4. 数分待つと本番に反映される（ブラウザで確認。表示が古い時はCtrl+F5）

### 公開を取り消したい（直前の状態に戻す）
Claude Codeに「サイトを1つ前の状態に戻して」と頼む（`git revert`で安全に戻せる。履歴は全部残っている）。

## 3. お問い合わせフォーム

### 初回セットアップ（未完了の作業）
1. https://web3forms.com を開く
2. 「info@anom-ai.com」を入力して Create Access Key
3. info@anom-ai.com に届いたメール内のアクセスキーをコピー
4. `assets/js/config.js` の `FORM_ACCESS_KEY: ""` の引用符の中に貼り付けて公開
5. **悪用防止**: Web3Formsの管理画面で Allowed Domains に `anom-ai.com` を登録（他人のサイトからキーを使われるのを防ぐ）

### 動作テスト（公開後・月1回推奨）
1. 本番サイトのフォームから自分でテスト送信（名前に「テスト」と書く）
2. info@anom-ai.com に届くことを確認
3. 届かない場合 → 迷惑メールフォルダ → Web3Formsのダッシュボード（送信ログ）の順に確認

### フォームの仕様（知っておくと安心）
- キー未設定の間は送信されず「メールでご連絡ください」と案内が出る（取りこぼし防止）
- 送信失敗時も自動でメールアドレスを案内する
- スパム対策済み（見えない欄に入力するボットは自動で弾く）

## 4. ドメインとDNS（初回接続の手順）

GitHub Pages側の設定はClaude Codeが実施済み。**ドメイン会社（レジストラ）側の設定だけ必要**:

1. anom-ai.com を買ったサービス（お名前.com / ムームー等）のDNS設定画面を開く
2. 以下のレコードを追加:

| 種別 | ホスト名 | 値 |
|---|---|---|
| A | @（空欄） | 185.199.108.153 |
| A | @（空欄） | 185.199.109.153 |
| A | @（空欄） | 185.199.110.153 |
| A | @（空欄） | 185.199.111.153 |
| CNAME | www | tapiai-coder.github.io |

3. 反映まで数分〜最大48時間（多くは1時間以内）
4. 反映後、GitHubのリポジトリ → Settings → Pages で「Enforce HTTPS」にチェック（鍵マーク必須。チェックできない場合は証明書発行待ちなので数時間後に再試行）

## 6. トラブルシューティング

| 症状 | 対処 |
|---|---|
| サイトが表示されない | GitHub → Settings → Pages が「main / (root)」になっているか確認。DNSを触った直後なら最大48時間待つ |
| 表示が崩れた | 直前の編集が原因。「1つ前に戻して」とClaude Codeに依頼 |
| フォームが届かない | 上記3.のテスト手順。キーが消えていないか config.js を確認 |
| 演出が重い・動かない端末がある | 仕様として自動対応済み（非対応端末は静的表示に自動切替・スマホは粒子数自動削減）。個別の不具合報告があれば端末名を控えてClaude Codeへ |
| 更新したのに反映されない | ブラウザのキャッシュ。Ctrl+F5（スマホは履歴から削除）。それでもダメならGitHubのActionsタブ/Pages反映待ち数分 |

## 7. 触ってはいけない場所

- `assets/vendor/` — サイトに同梱した外部ライブラリ。**消す・上書きする・CDNに戻すのは禁止**（動作検証済みのバージョンで固定中）
- `index.html` 冒頭の `Content-Security-Policy` の行 — セキュリティの要。外部サービスを追加する時はClaude Codeに依頼
- `archive/astro-v3` ブランチ — 旧サイトの保管庫。削除しない
